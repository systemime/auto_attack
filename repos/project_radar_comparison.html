深度解析（8 个比较点）

1. DAST 闭环

AutoAttack 已形成侦察、指纹、扫描、验证、证据和报告闭环；nuclei、sqlmap、subfinder、amass 是更强的单点组件，适合接入为 skill，而不是替代主流程。

2. 安全门控

AutoAttack 的 scope/policy/approval/audit 是当前最该保留的护城河；caldera、osmedeus、sqlmap 等能力更强但执行面更宽，默认不应绕过审批和范围约束。

3. Agent 编排

LuaN1aoAgent、Pentest-Swarm-AI、PentestGPT、watchtower 在 planner/worker/analyst 叙事上更激进；AutoAttack 应吸收可解释路由和任务拆分，不必引入重型认知图谱。

4. 分布式执行

osmedeus 与 Pentest-Swarm-AI 的工作流、worker、云端执行思想值得借鉴；AutoAttack 当前优势是 SQLite/Redis queue、resume 和轻量恢复，扩展时仍应保持任务状态可审计。

5. 知识资产

Arsenal 与 nuclei-templates 的 payload、模板、checklist、命令片段是最可吸收资产；更适合转为 metadata、playbook 或只读知识索引，不建议复制大型内容库。

6. 专项深度

sqlmap 的 SQLi、amass/subfinder 的资产发现、nuclei 的模板扫描、agentic_security 的 LLM 安全都明显更深；AutoAttack 应承认专项差距，并通过适配器组合它们。

7. 轻量可审计

AutoAttack 相比 Strix、autocve、airecon、caldera 更轻，部署和审计路径更短；本轮以 deep profile、增强 parser、覆盖矩阵和轻量 dashboard 补强，仍避免引入完整平台或新前端状态模型。

8. 产品体验

Arsenal、osmedeus、Strix、Nettacker 在 UI/API/报告体验上有参考价值；AutoAttack 已补 dashboard、过滤、发现详情、暂停刷新和 /api/summary，仍不重写成大型 Web 平台。
